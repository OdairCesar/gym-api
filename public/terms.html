1. Aceitação dos Termos

Ao acessar ou utilizar a Gym API — seja por meio de integração direta com a API REST, por meio de aplicações construídas sobre ela ou por meio desta interface —, você concorda em vincular-se a estes Termos de Uso e à nossa Política de Privacidade.

Se você representa uma organização (academia), ao aceitar estes termos, você declara possuir autoridade para vincular tal organização. Neste caso, "você" refere-se tanto ao indivíduo quanto à organização representada.

Atenção: Se não concordar com qualquer parte destes termos, não utilize a plataforma.

2. Descrição do Serviço

A Gym API é uma plataforma de software no modelo SaaS que oferece:

  • API RESTful multi-tenant para gestão de academias
  • Sistema de gerenciamento de usuários com 4 níveis de acesso (Super, Admin, Personal, Cliente)
  • Criação e gerenciamento de dietas personalizadas (refeições e alimentos)
  • Criação e gerenciamento de treinos personalizados (exercícios)
  • Controle de estoque e produtos da academia
  • Sistema de permissões cross-tenant para colaboração entre academias
  • Planos de assinatura com diferentes limites de recursos

O serviço é disponibilizado "no estado em que se encontra" durante o período de MVP (Minimum Viable Product). Funcionalidades podem ser adicionadas, alteradas ou removidas conforme o desenvolvimento do produto.

Novos usuários são criados com status pendente de aprovação. O acesso completo está condicionado à aprovação por um administrador ou personal da academia. Reservamo-nos o direito de suspender contas que violem estes termos.

3.3 Super Users

Contas com nível "Super User" são criadas exclusivamente via banco de dados por administradores do sistema e não podem ser solicitadas por usuários comuns via API, por razões de segurança.

Planos pagos são cobrados mensalmente no início de cada ciclo de faturamento. Os pagamentos são processados por provedores terceirizados (Google Pay, Apple Pay) e sujeitos aos termos desses provedores.

5.3 Upgrades e downgrades

Você pode alterar seu plano a qualquer momento. Upgrades têm efeito imediato. Em caso de downgrade, o novo limite de usuários será aplicado no próximo ciclo — usuários excedentes não serão excluídos automaticamente, mas a criação de novos usuários ficará bloqueada até que o número esteja dentro do limite.

5.4 Cancelamento e reembolso

O cancelamento encerra o acesso ao plano ao final do período vigente. Não há reembolso proporcional para cancelamentos no meio do ciclo, salvo determinação legal em contrário ou acordo expresso.

Plano gratuito: Academias que não efetuarem upgrade permanecem no plano inicial indefinidamente, sujeitas ao limite de 25 usuários.
